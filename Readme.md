## ブランチについて
主な開発ブランチはdevelopにする予定です。 
コンフリクトを防ぐためにdevelopブランチには直接コミットしないことを推奨します。
mainとdevelopの間ではプルリクでの厳密なコードレビューを行ったあと、マージされます。
図に示すと、
<pre>
main
└── develop
    └── some branches(ui,scripting,updatingなど)
</pre>
